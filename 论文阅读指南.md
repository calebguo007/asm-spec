## 二、每篇论文的定制阅读策略

### 总体原则

**不要从 Abstract 一直读到 Conclusion。** 用这个流程：

```
1. 读 Abstract（1 分钟）→ 确认和 ASM 的关系
2. 看 Figure 1（通常是架构图）→ 快速理解全貌
3. 跳到你需要回答的具体问题对应的章节 → 精读
4. 看 Conclusion + Limitations → 找 ASM 可以填补的空白
5. 记录 2-3 条可直接引用的句子
```

---

### 论文 1：RouteLLM（30min）

**你要回答的问题**：
1. ASM Scorer 和 RouteLLM 的路由算法有什么本质区别？
2. RouteLLM 的 cost-quality tradeoff 是怎么量化的？
3. ASM 怎么和 RouteLLM 互补？

**阅读顺序**：
1. **Abstract**（1min）— 确认它只做 LLM 品类内路由
2. **Section 3: Router Design**（10min）— 看 4 种路由算法的输入输出，特别是矩阵分解那个
3. **Section 4: Evaluation / Figure 3-4**（10min）— 看 cost vs quality 的 Pareto 曲线图，理解它怎么量化 tradeoff
4. **Section 6: Limitations**（5min）— 找它不能做什么（跨品类选择？非 LLM 服务？）
5. **记录**：摘 2 句话用于论文 Related Work

**不需要读**：Section 2 (Background)、Section 5 (Ablation details)

---

### 论文 2：Dynamic Model Routing Survey（45min）

**你要回答的问题**：
1. 路由系统的完整分类法是什么？ASM 在哪个位置？
2. "何时决策、用什么信息、怎么计算"的三维框架怎么用？
3. 有没有涉及"跨品类"路由的工作？

**阅读顺序**：
1. **Abstract**（2min）— 看分类维度
2. **Table 1 / Figure 1**（5min）— 一定有分类总表或架构图，这是全文最有价值的东西
3. **Section 2: Taxonomy**（15min）— 精读分类法，标记每个类别和 ASM 的关系
4. **Section 3: 挑"Cascading"部分读**（10min）— 级联和 ASM 的两阶段（Filter→Rank）最相关
5. **Section 5: Challenges / Future Directions**（10min）— 找"跨品类""经济学""定价"相关的 gap
6. **记录**：摘三维框架的定义 + 1-2 句 future direction 中和 ASM 相关的

**不需要读**：每种路由方法的具体算法细节（太多了，不是你的研究方向）

---

### 论文 3：AI Agent Protocol Survey（30min）

**你要回答的问题**：
1. 二维分类法的具体定义是什么？
2. 现有协议的 gap analysis 里有没有提到"经济学/定价"？
3. 未来方向里有没有 ASM 可以切入的点？

**阅读顺序**：
1. **Abstract**（2min）— 确认二维分类
2. **Figure 1 / Table 1**（5min）— 分类矩阵图
3. **Section 3: 找 MCP 和 A2A 的分析段落**（10min）— 看它怎么描述 MCP 的局限性
4. **Section 5: Gap Analysis / Challenges**（10min）— 找"economic""pricing""marketplace""service selection"关键词
5. **Section 6: Future Directions**（5min）— 找 ASM 能引用的"需要做什么"
6. **记录**：摘分类法的定义 + gap analysis 中和 ASM 相关的 1-2 句话

**不需要读**：每个具体协议（ACP、ANP 等）的技术细节

---

### 论文 4：Pay for the Second-Best Service / WWW 2026（40min）

**你要回答的问题**：
1. Provider 的两种造假行为（偷梁换柱 + 注水）的形式化定义是什么？
2. 近似激励相容机制是怎么设计的？核心直觉是什么？
3. ASM + Signed Receipts 怎么对接这个机制？

**阅读顺序**：
1. **Abstract**（2min）— 确认问题定义和解法
2. **Section 1: Introduction**（5min）— 看 Figure 1（通常有用户-provider 交互模型图）
3. **Section 2: Problem Formulation**（10min）— 精读！这是你需要引用的核心数学定义
4. **Section 3: Mechanism Design**（15min）— 看机制的核心直觉（不用推导每一步数学），重点看 Theorem 1 的含义
5. **Section 6: Discussion / Limitations**（5min）— 找"不解决什么"
6. **记录**：摘问题定义 + Theorem 1 的自然语言描述 + 1 句 limitation

**不需要读**：Section 4-5 的完整数学证明（除非你要做理论扩展）

---

### 论文 5：AaaS-AN（25min）

**你要回答的问题**：
1. RGPS 标准的四个维度具体是什么？
2. 它的"服务发现+注册"和 ASM Registry 有什么区别？
3. 怎么证明 ASM 和 AaaS-AN 是互补不是竞争？

**阅读顺序**：
1. **Abstract**（2min）
2. **Section 2 / Figure 1: Architecture**（10min）— 看整体架构图，标记和 ASM 的交叉/互补
3. **找"Service Discovery"或"Service Registration"相关段落**（10min）— 对比 ASM Registry
4. **Conclusion**（3min）— 看它自己认为的局限
5. **记录**：摘 RGPS 定义 + 1 句互补关系的论据

**不需要读**：数学推理/代码生成的实验细节

---

### 论文 6：MCP Landscape & Security（20min）

**你要回答的问题**：
1. MCP 生态的规模数据（多少 server、多少用户、增长速度）？
2. 安全威胁分类中，哪些和 ASM 的信任机制相关？
3. 未来方向中有没有提到"经济学/定价"？

**阅读顺序**：
1. **Abstract**（1min）
2. **Section 2: MCP Landscape**（10min）— 精读，抓生态数据
3. **Table X: Threat Taxonomy**（5min）— 扫一遍，标记和 ASM 信任相关的
4. **Section 6: Future Directions**（5min）— 找 gap
5. **记录**：摘 2-3 个生态数据 + 1 句 future direction

**不需要读**：具体攻击案例的技术细节

## 四、推荐阅读顺序（周末 2 个半天）

### 周六上午（3h）
1. RouteLLM（30min）→ 搞清 ASM vs LLM Routing 的区别
2. Dynamic Routing Survey（45min）→ 建立路由领域的全景认知
3. 休息 15min
4. AI Agent Protocol Survey（30min）→ 确认 ASM 在协议生态中的定位
5. 整理笔记（30min）→ 填产出模板

### 周六下午或周日（2h）
6. Pay for the Second-Best Service（40min）→ 信任机制的理论根基
7. AaaS-AN（25min）→ 确认和最近竞品的差异
8. MCP Security（20min）→ 补充生态数据
9. 整理笔记 + 更新调研文档（30min）
